RNF 04 - Velocidad de Busqueda
Resultado: Cumple
Descripcion de la Verificacion: Se realizaron 10 peticiones GET a los endpoints de busqueda /api/products?name=notebook y /api/categories?name=electronica, midiendo el tiempo de respuesta con Postman. El tiempo promedio fue de 0.3 segundos, muy por debajo del limite de 3 segundos establecido en el ERS. Esto se debe a que el sistema utiliza una base de datos H2 en memoria con indices JPA y el volumen de datos de prueba es reducido.

---

RNF 05 - Datos Sensibles Cifrados
Resultado: No Cumple
Descripcion de la Verificacion: Se reviso la configuracion de seguridad del backend. El sistema utiliza NoOpPasswordEncoder de Spring Security, el cual NO aplica ningun tipo de cifrado ni hashing a las contrasenas. Las contrasenas se almacenan en texto plano en la base de datos H2. Ademas, la comunicacion via HTTP (sin HTTPS) no cifra los datos en transito. Para cumplir con este requisito seria necesario implementar BCryptPasswordEncoder o similar, y configurar HTTPS.

---

RNF 06 - Interfaz Intuitiva y Facil de Usar
Resultado: No Cumple
Descripcion de la Verificacion: La aplicacion actualmente es solo un backend de API REST sin interfaz de usuario (frontend). El ERS menciona Angular como framework planificado "(a agregar)", lo que indica que el frontend aun no ha sido desarrollado. No es posible evaluar la intuicion o facilidad de uso de la interfaz sin un frontend implementado.

---

RNF 07 - Verificacion de Edad
Resultado: No Cumple
Descripcion de la Verificacion: Se reviso el flujo de registro de usuarios en el backend. El sistema permite registrar usuarios con cualquier email y rol sin realizar ninguna verificacion de edad. No existe integracion con Gmail ni con ningun otro servicio para validar la mayoria de edad del usuario. El campo "role" se asigna directamente desde la solicitud sin validacion adicional.
