======================================================================
 POSTMAN TESTING GUIDE - Generic E-Commerce API
 Flujo completo: Registro -> Categorias -> Productos -> Carrito -> Orden
======================================================================

BASE URL: http://localhost:8080
AUTH: HTTP Basic (excepto register/login y GETs publicos)

La mayoria de endpoints protegidos requieren:
  - Authorization: Basic <base64(email:password)>
  - X-User-Email: <email>  (en cart, order, review)

======================================================
 PASO 1: Registrar usuario ADMIN
======================================================
POST {{BASE_URL}}/api/users/register
Content-Type: application/json

{
  "email": "admin@tienda.com",
  "password": "admin123",
  "name": "Admin Principal",
  "role": "ADMIN"
}

---
 Respuesta esperada: 201 Created
{
  "id": 1,
  "email": "admin@tienda.com",
  "name": "Admin Principal",
  "role": "ADMIN"
}

======================================================
 NOTAS IMPORTANTES
======================================================
1. El header X-User-Email es obligatorio en endpoints de
   carrito, ordenes y reviews. Sin él da error 400.

2. Spring Security usa HTTP Basic con NoOpPasswordEncoder
   (sin encriptacion).

3. La base de datos H2 es en memoria: los datos se pierden
   al reiniciar la aplicacion.

4. Para iniciar:
   cd Generic-E-Commerce
   ./mvnw.cmd spring-boot:run
